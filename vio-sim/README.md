# Модули кода

```
vio_data.py        источник VIO: модель дрейфа
vio_correction.py  модуль коррекции: offset + отбивка
vio_bridge.py      мост: истина -> модель -> коррекция -> MAVLink; origin/home
vio_launch.py      окно 2: сборка модулей + консоль оператора; весь CONFIG
```

---

## vio_data.py — источник VIO (модель дрейфа)

Моделируются не сырые датчики, а **их итоговый эффект на выходе одометрии** -
остатки ошибок, которые внутренний фильтр реального VIO не смог доесть.
Дрейфуют только ненаблюдаемые степени свободы (X, Y, Z, курс); **крен и тангаж
поправляются постоянно направлением акселерометра** - не моделируются.

Паспорт переменных:

| Переменная | Что это | Единица |
|---|---|---|
| `a_ost` | остаток ошибки акселерометра в текущем тике (каждый тик новый, сам не копится) | м/с² |
| `v_err` | накопленная ошибка скорости — «ложная скорость» | м/с |
| `p_err` | накопленный снос позиции | м |
| `w_ost` | остаток ошибки гироскопа по курсу | град/с |
| `psi_err` | накопленная ошибка курса | рад |

Тик (30 Гц, `dt = 1/30`):

```
a_ost   = N(0, σ_a)            строка 1: «кубик» — остаток акселя
v_err  += a_ost * dt           строка 2: интеграл N1 (ускорение -> скорость)
v_err  *= 1 - dt/τ             строка 2а: затухание ложной скорости (τ = 60 с)
p_err  += v_err * dt           строка 3: интеграл N2 (скорость -> позиция)
w_ost   = N(0, σ_w)            строка 4: остаток гироскопа (ось курса)
psi_err += w_ost * dt          строка 5: интеграл (угл. скорость -> угол)
p_vio   = Rz(psi_err) @ p_true + p_err + N(0, σ_n)     строка 6
yaw_vio = yaw_true + psi_err                            строка 7
```

Строка 2а — тормоз интеграла №1: реальный VIO хорошо видит **скорость**
(кадр-к-кадру), поэтому ложная скорость не может разгоняться бесконечно —
с τ = 60 с она гуляет в пределах ~±0.3 м/с. Позиция при этом дрейфует
неограниченно (как у настоящего VIO), но линейно, а не с разгоном.

Строка 6 по кускам: поворот `Rz` — ошибка курса разворачивает **всю траекторию**
вокруг старта (вклад в позицию ∝ удалению: 1° на 100 м = 1.75 м вбок); `+ p_err` —
снос, который копился во времени и от траектории не зависит; `+ N(0, σ_n)` —
свежая дрожь камеры, не копится. Строка 7: курс врёт **тем же** углом, что
повёрнута позиция, — источник ошибается согласованно, как настоящий.

`seed` фиксирует генератор: «случайный» полёт воспроизводим.

## vio_correction.py — модуль коррекции

Дрейф — свойство источника, лечится в потоке: `offset` вычитается из **каждого**
выходного сообщения.

- `apply(p)` — вычесть offset, запомнить отправленное (основа для snap);
- `snap(name)` — отбивка: `Δ = последнее_отправленное − маркер; offset += Δ`.
  Δ считается от уже исправленного выхода (а не от исходного значения). Промах оператора остаётся
  остаточной ошибкой: не копится, снимается следующей отбивкой;
- `bump()` — `reset_counter = (reset_counter + 1) % 256` (байт MAVLink): пометка
  «скачок потока легален».

## vio_bridge.py — мост в автопилот

Конвейер каждого тика:

```
SIMSTATE (истинные lat/lng/углы) -> метры NED от origin -> model.tick()
    -> correction.apply() -> VISION_POSITION_ESTIMATE (30 Гц)
```

Ключевые места:

- **connect()** — терпеливо ждёт Heartbeat (SITL может ещё собираться); запрашивает
  телеметрию (без запроса SITL на порту молчит); представляется **sysid=1,
  compid=197** («компонент визуально-инерциальной одометрии») — как бортовой
  VIO-модуль; кастомный Mission Planner принимает VIO-сообщения только от sysid борта;
- **setup_origin()** — origin из первого SIMSTATE; `SET_GPS_GLOBAL_ORIGIN`
  с повторами до подтверждения-эха (EKF принимает origin только после
  инициализации фильтра); затем home через `COMMAND_INT / DO_SET_HOME` с проверкой
  ACK (без home коптер не армится);
- **ковариация** — упаковка 21 числа (верхний треугольник 6×6), заполняется
  диагональ: дисперсии позиции = σ_n², крен/тангаж малые (не дрейфуют), курс ±3°;
- **крен/тангаж отправляются истинными** — у VIO они не дрейфуют (гравитация — якорь);
- **snap/loss/status** — команды для консоли; loss делает `bump()` — возврат
  потока после паузы принимается без боя с гейтом EKF: посылается reset_counter, сообщающий контроллерам, что не нужно совершать резкий рывок, что дрон не телепортировался и нет огромной ошибки, произошла штатная коррекция. Иначе EKF будет гейтом отклонять долго коррекцию, потом сдастся и примет, а контроллер получит резкий скачок ошибки и полетит на точку со срывом.
- защита от смерти SITL: 5 секунд без истины (SIMSTATE) → выход (не зависаем в EOF-спаме).

## vio_launch.py — окно 2

Сборка трёх модулей + консоль оператора. Никакой предметной логики: прочитал
CONFIG → создал объекты → connect/setup_origin → поток `bridge.run()` → пересылка
команд консоли. Все параметры стенда (связь, модель, маркеры) — в одном блоке
CONFIG в этом файле.

## vio_params.parm — параметры автопилота

Грузятся при старте SITL (`--defaults`), reboot не нужен. Ключевое:
`GPS1_TYPE=0` (без GPS), `VISO_TYPE=1` (MAVLink-бекенд), `EK3_SRC1_POSXY/YAW=6`
(позиция и курс — внешняя навигация), `EK3_SRC1_POSZ=1` (высота — баро),
`EK3_SRC1_VELXY/VELZ=0` (скорости не подаём — источники «нет», иначе дефолт GPS
валит арминг), `COMPASS_USE*=0` (один «север» с VIO).
